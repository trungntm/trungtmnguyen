## Bộ nhớ container không phải là heap

JVM nhìn thấy nhiều vùng bộ nhớ khác nhau: heap, metaspace, native buffers, thread stacks và phần
overhead của JIT. Trong khi đó Kubernetes chỉ nhìn thấy giới hạn cgroup, vì vậy container vẫn có
thể bị kill ngay cả khi heap trông vẫn ổn.

### Checklist cơ bản

1. Thiết lập CPU và memory requests sát với thực tế.
2. Giới hạn heap một cách tường minh để hành vi dễ dự đoán hơn.
3. Chừa đủ headroom cho các vùng cấp phát ngoài heap.

```bash
JAVA_TOOL_OPTIONS="-XX:MaxRAMPercentage=70 -XX:InitialRAMPercentage=50"
```
