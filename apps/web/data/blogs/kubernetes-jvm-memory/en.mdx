## Container memory is not the heap

The JVM sees several memory regions: heap, metaspace, native buffers, thread stacks, and JIT
overhead. Kubernetes only sees the cgroup limit, so a container can be killed even when the heap
looks healthy.

### Baseline checklist

1. Set realistic CPU and memory requests.
2. Cap heap size explicitly for predictable behavior.
3. Leave headroom for non-heap allocations.

```bash
JAVA_TOOL_OPTIONS="-XX:MaxRAMPercentage=70 -XX:InitialRAMPercentage=50"
```
