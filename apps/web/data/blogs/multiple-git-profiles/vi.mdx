Nếu bạn chưa biết cách tạo `ssh-key`, hãy tham khảo bài [creating-ssh-key](https://trungtmnguyen.com/blog/creating-ssh-key).

## Giới thiệu

Trong trường hợp tôi có nhiều `ssh-key` và muốn dùng chúng cho từng Git repo cụ thể, thay vì sao
lưu rồi cập nhật thư mục `.ssh` thủ công mỗi lần, tôi thích dùng một shell script nhỏ để tự động
hóa việc này.

## Bắt đầu

Hãy nhìn vào thư mục `.ssh` của bạn.

- Tôi sẽ tạo một thư mục mới tên `trungntm` để chứa `ssh-key` của user trungntm
- Tôi cũng tạo thêm một thư mục tên `developer1` để chứa `ssh-key` của user developer1
- Bước tiếp theo, bạn cần di chuyển thủ công `id_rsa` và `id_rsa.pub` vào các thư mục mới này. _Lưu ý: hãy kiểm tra đúng file trước khi di chuyển để đảm bảo bạn dùng đúng key_
- Sau bước này, bạn đã có đủ file trong các thư mục mới. Hãy mở `.bash_profile` và cập nhật như bên dưới:

```
cd ~
vi .bash_profile
```

Cập nhật hàm `shell` sau vào `.bash_profile`. _Đừng quên thay đổi đường dẫn user trên máy của bạn cho phù hợp_

```Bash
function use-ssh() {
    echo "🚀 🚀 🚀 Switching to ssh profile" $1
    cp /Users/trung.nguyen/.ssh/$1/id_rsa.$1 /Users/trung.nguyen/.ssh/id_rsa
    cp /Users/trung.nguyen/.ssh/$1/id_rsa.pub.$1 /Users/trung.nguyen/.ssh/id_rsa.pub
    echo "Completed 🔥 ✅"
}
```

- Lưu lại, mở Terminal mới và thử lệnh `use-ssh`:

```bash
use-ssh trungntm
```

Lệnh này sẽ được gọi và cập nhật `id_rsa` cùng `id_rsa.pub` thành các file chính trong thư mục `.ssh`, kết quả sẽ tương tự như sau:

```bash
trung.nguyen@trungtmnguyen ~ % use-ssh trungntm
🚀 🚀 🚀 Switching to ssh profile trungntm
Completed 🔥 ✅
```

Hãy thử chuyển sang `developer1`

```bash
trung.nguyen@trungtmnguyen ~ % use-ssh developer1
🚀 🚀 🚀 Switching to ssh profile  developer1
Completed 🔥 ✅
```

Vậy là xong. Bây giờ bạn có thể tạo nhiều profile và chuyển đổi giữa chúng dễ dàng bằng shell command.
